# CLAUDE.md — Project instructions

Durable project rules for AI agents. **`AGENTS.md` is generated by Laravel Boost**
(`boost:update` runs on composer operations and overwrites it) — do NOT hand-edit
`AGENTS.md`; put lasting guidance here or in `docs/`.

## Code standards (enforced)

- **Frontend (JS / TS / TSX / JSON): Biome.** Run `bun run check`
  (`biome check --write .`) before finalizing any frontend change — it formats,
  organizes imports, and applies safe lint fixes. Verify with `bun run check:ci`
  (must be **0 errors**). Biome **replaced ESLint + Prettier** — do not
  reintroduce them. Full policy: [`docs/CODING-STANDARDS.md`](docs/CODING-STANDARDS.md).
- **UI / UX: shadcn + branded, not plain forms.** All UI must use the design
  tokens (indigo brand + neutral surfaces, light **and** dark), reach for
  shadcn/ui components, and handle every state (loading / empty / error / success)
  with real feedback. Never ship a bare black-and-white form. Full policy:
  [`docs/UI-UX-GUIDELINES.md`](docs/UI-UX-GUIDELINES.md)
  (agent digest: [`.ai/guidelines/ui-ux.md`](.ai/guidelines/ui-ux.md)).
- **Vendored shadcn/ui is READ-ONLY.** Never hand-edit
  `resources/js/components/ui/**` (or other generated trees:
  `resources/js/{routes,actions,wayfinder}`, `bootstrap/ssr`, `resources/css`).
  Editing a primitive to fix one screen reliably breaks others. Instead: pass
  props/className, **wrap** the primitive in your own `components/` file, or
  compose `radix-ui` directly — never fork `ui/`. Full rule:
  [`docs/CODING-STANDARDS.md`](docs/CODING-STANDARDS.md).
- **PHP: Laravel Pint.** Run `vendor/bin/pint --dirty` before finalizing PHP changes.
- **Types:** `bun run types:check` must pass.
- **Package manager / JS runtime: Bun** (`bun install`, `bun run …`; lockfile
  `bun.lock`). Not npm / pnpm / yarn.

## Testing

- Pest. Run `php artisan test --compact` (or `--filter=Name`). This project is
  built TDD per the plans in `docs/superpowers/plans/` — write the failing test
  first, then implement.

## Architecture

See `docs/superpowers/specs/` (master spec) and `docs/superpowers/plans/`.

- Multi-tenant manufacturing/inventory ERP: Laravel 13 + Inertia v3 + React 19 +
  TypeScript, Tailwind v4. `stancl/tenancy` in **multi-database** mode,
  tenants identified by **URL slug** (`/{slug}/…`).
- One central (landlord) DB (super-admins + `organizations`) + one DB per tenant
  (that tenant's `users` + business tables; no `organization_id` columns).
- The central DB connection is named **`central`**. **Never** define a connection
  literally named `tenant`.
- Central migrations live in `database/migrations/`; per-tenant migrations in
  `database/migrations/tenant/`. Keep them straight.

## Pre-finalize checklist

1. `bun run check` (frontend changed) · `vendor/bin/pint --dirty` (PHP changed)
2. `bun run types:check`
3. `php artisan test --compact` for affected tests
